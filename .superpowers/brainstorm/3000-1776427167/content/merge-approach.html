Live Feed & Session Overview — 合并方案

目前 Live Feed 是原始事件流，Session Overview 是聚合卡片，两者功能重叠。你想合并成一个更有意义的页面。先看以下三个方向：

A

统一 Session 页面 — 时间线 + 详情

左侧 Session 列表（可点击），右侧是该 Session 的时间线视图（带过滤/展开）。点击 KPI 上的 "Sessions Active" 跳转到这个统一页面。

B

Dashboard 图形化 — 保留两个页面

Dashboard 的 Sessions Active 用可视化图表展示（柱状图/热力图），但 Live Feed 和 Session Overview 保持独立。KPI 点击各自跳转。

C

统一 Sessions 页面 — 双层视图

顶部：聚合指标图表（每小时事件数、活跃 Session 趋势）。中间：Session 列表卡片。点击任一 Session 展开其事件时间线。一个页面包含所有信息。

推荐：方案 C

  • Dashboard 的 Sessions Active 跳转这个统一页面
  • 移除 Session Timeline 侧边栏入口（已删除）
  • Live Feed 的原始事件流被整合为 Session 时间线子视图
  • 数据更直观：先看趋势，再看具体 Session

Trade-off

  • 页面比现有任何一个都大，需要好的折叠/过滤
  • 需要新增一个聚合指标 API（每 N 分钟事件数）
